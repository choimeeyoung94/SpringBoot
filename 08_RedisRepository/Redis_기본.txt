1. Redis
  1) Remote Dictionary Server
  2) 인메모리(In Memory) 기반의 오픈소스 NoSQL 데이터베이스입니다.
  3) 데이터를 디스크가 아니라 시스템 메모리에 저장하기 때문에 읽기/쓰기가 매우 빠릅니다.

2. 기본 개념
  1) Key-Value 구조를 기반으로 한 비관계형 데이터베이스입니다.
  2) NoSQL이므로 SQL 대신 명령어 기반 API로 데이터를 관리합니다.
  3) 캐시, 임시 저장, 세션 저장소 등 다양한 용도로 활용할 수 있습니다.
  4) String, List, Set, Hash, Sorted Set 등의 자료 구조를 지원합니다.
  5) Single Thread 방식으로 동작합니다(데이터의 일관성 및 동시 접근 제어가 용이).

3. RedisRepository
  1) Spring Data Redis가 제공하는 Repository 인터페이스 구현체입니다.
  2) CRUD 중심의 추상화를 제공합니다.
  3) JPA Repository와 유사한 CRUD 메소드를 제공합니다.
  4) 간단한 CRUD 작업에 적합합니다.

4. RedisTemplate
  1) Redis의 다양한 기능을 사용할 수 있는 Repository 추상화 도구입니다.
  2) 세밀하게 Redis 명령어를 조작해서 복잡한 기능을 구현할 수 있습니다.
  3) Redis 전체 기능을 세밀하게 제어할 수 있으므로, 복잡한 Redis 활용 및 최적화가 필요한 경우에 적합합니다.


5 Windows에서 Redis 설치 및 실행
  1) Redis는 공식적으로 제공되는 Windows용 바이너리가 없습니다 (2025년 현재).
  2) Microsoft Open Tech가 개발한 포크 버전 사용 가능(비공식):
    (1) Memurai
      또는
    (2) https://github.com/tporadowski/redis/releases 에서 최신 Windows용 Redis 다운로드 (수업에서 사용한 방식)
  3) ZIP 파일 다운로드 후 압축 해제합니다.
  4) 압축 해제 폴더에서 redis-server.exe 실행 파일 더블 클릭 또는 명령 프롬프트에서 실행합니다.
  5) 별도 설정이 없으면 기본 포트 6379에서 실행되는 걸 확인할 수 있습니다.
